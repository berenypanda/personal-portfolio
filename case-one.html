Description

Details

A semester-long team project for my Internet Marketing class.

Objective

  • To apply Internet Marketing concepts to build a simple website for a hairstylist (Alexus Hair Designs.)
  • To create PHP supported interactive features that would support the business needs of Alexus Hair Designs.

Journey

Choosing a Client

Our client was my team member’s hairstylist, Alexus. Alexus is a traveling hairstylist that performs services at clients’ homes. My partner personally knowing the client gave us a better idea of the user journey that the website should support.

Analyzing The Client

  • We created a business model of Alexus Hair Designs and conducted a SWOT Analysis to determine what features to place on the website.
    • Alexus’s value proposition, or main benefit to prospective clients, is the convenience of using a hairstylist in the comfort of one’s own home.
    • The convenience gives Alexus a competitive advantage, but customers may turn to salons for familiarity and lower prices in sacrifice of convenience.
  • Observing competitors reminded us to have transparent pricing and to use images to advertise to our known demographic (mainly 25-60 year old women.)

Designing and Building

  • My partner translated the brand image and selling points into parts of our design.
    • Color Palette and Typography
      • Soft neutral colors and sans serif fonts helped create a sleek, professional and relaxing overall feel to the website.
    • Call To Action and Hero Image
      • Soft neutral colors and sans serif fonts helped create a sleek, professional and relaxing overall feel to the website.
  • I focused on building the two main interactive features using PHP.
    • Booking Appointment Form
      • This form supports the main call to action. It also generates a basic confirmation email using the PHP mail() function. I had an interest in creating a stylized confirmation email, but this proved too complex for me at the time.
    • Promo Code
      • The promotional message at the top of the website changes the code based on the time of day.

Challenges

  • Viewing changes in real time became tedious with the UTA Cloud hosting service. I’d have to reupload my file each time just to see tiny changes I made in the CSS, for example.
    • I solved this problem by discovering MAMP, a tool that allows me to run a server locally to play around with dynamic websites. This allowed me to view changes in real time, speeding up my development process.
  • While I was experienced in using git version control as an individual, using git with another person became challenging.
    • We decided to split our work beforehand each day, with me sending files at the end of the day.

Conclusion

What I'd Do Differently

  • Utilizing more real images from the client, especially those of high quality.
  • Utilize git as a team instead of sending files asynchronously. I learned that through GitHub desktop and VSCode, there are ways to commit and push changes without everyone having to learn bash language.
  • Place the main call to action on several pages. I noticed that it is only on one page, which forces the user to return to the homepage. Ideally, users should be able to explore services, or learn about the client, then be reminded with a nearby button on the same page to perform the main objective (booking an appointment.)
